### explanation from clade(I have to be honest in this place : )


The new instruction that appeared compared to `declarationv1` is:

```asm
111d:  c7 45 fc 64 00 00 00    movl   $0x64,-0x4(%rbp)
```

This is the compiler storing your value onto the stack. `$0x64` = 100 decimal. `-0x4(%rbp)` is the address 4 bytes below the base pointer — that slot **is** `a`. Four bytes because `int` is 32 bits.

Also notice the `.fini` section shifted — it's now at `0x112c` instead of `0x1124`, because `main` is 2 bytes longer than before.

What's still missing is a `sub $0x4,%rsp` — GCC didn't bother adjusting the stack pointer for a single small local. It just used the space below `%rbp` directly.


### Minimal enough for basic understanding of diffs 

| Instruction | Meaning |
|---|---|
| `push %rbp` | Save a register onto the stack |
| `pop %rbp` | Restore a register from the stack |
| `mov %rsp,%rbp` | Copy one register's value into another |
| `mov $0x0,%eax` | Put a literal value into a register |
| `movl $0x64,-0x4(%rbp)` | Write a literal value to a memory address |
| `sub $0x8,%rsp` | Subtract from a register (here, moves stack pointer down — allocates space) |
| `add $0x8,%rsp` | Add to a register (here, moves stack pointer back up — frees space) |
| `call *%rax` | Jump to a function whose address is in a register |
| `ret` | Return from the current function |
| `xor %ebp,%ebp` | XOR a register with itself — result is always zero (fast way to zero a register) |
| `lea 0xda(%rip),%rdi` | Compute an address and store it in a register (no memory read) |
| `test %rax,%rax` | Check if a register is zero (sets flags, no result stored) |
| `je` | Jump if equal / jump if zero flag is set |
| `jmp *%rax` | Unconditional jump to address in a register |
| `hlt` | Halt the CPU |
| `nop` / `nopw` | Do nothing — padding for alignment |
| `endbr64` | Security hint: marks a valid branch target (Intel CET) |
| `cmp` | Compare two values by subtracting, sets flags, discards result |

### Some terms : not necessary for now, but knowing is good 

Registers are small storage slots **inside the CPU itself** — not RAM, not stack. The CPU uses them to hold values it is actively working with.

x86-64 has 16 general-purpose registers:

| Register | Full name | Size | Common use |
|---|---|---|---|
| `%rax` | Accumulator | 64-bit | Return values, arithmetic |
| `%rbx` | Base | 64-bit | General purpose |
| `%rcx` | Counter | 64-bit | Loop counters |
| `%rdx` | Data | 64-bit | Arithmetic, function args |
| `%rsi` | Source Index | 64-bit | Function arg 2 |
| `%rdi` | Destination Index | 64-bit | Function arg 1 |
| `%rsp` | Stack Pointer | 64-bit | Points to top of stack |
| `%rbp` | Base Pointer | 64-bit | Points to base of current stack frame |
| `%r8`–`%r15` | Extended | 64-bit | Extra function args, general use |
| `%rip` | Instruction Pointer | 64-bit | Points to next instruction to execute |

The `r` prefix means 64-bit. The same register has smaller views:

| Name | Bits | Example |
|---|---|---|
| `%rax` | 64 | full register |
| `%eax` | 32 | lower half |
| `%ax` | 16 | lower 16 bits |
| `%al` | 8 | lowest byte |

That's why you see both `%rax` and `%eax` in the same disassembly — same physical register, different width access.
