Discovery

int main() vs int main(void)

main() means "accepts unspecified arguments." main(void) explicitly means none. The -Wpedantic flag catches this. Modern C always uses int main(void).

Discovery

Why does missing return 0 still work?

In C99 and later the compiler inserts return 0; implicitly at the end of main if omitted. This only applies to main — no other function gets this treatment. Explicit is always better.

Error Documented → errors/helloworld.c.error

Return type defaults to int

compiler output 
// broken — missing int before main
main() {
helloworld.c:6:1: error: return type defaults to 'int' [-Wimplicit-int]
    6 | main() {
      | ^~~~

Old K&R C allowed omitting the return type — implicit int. Modern C with -Wall flags this as an error. Always write int main(void).

1.2 — Fahrenheit → Celsius

K&R Chapter 1 table program. Five versions, three bugs, two big discoveries.

int variables while loop format specifiers integer division undefined behavior
Final Solution → fartocelv4.c

Multiply before dividing

fartocelv4.c 
#include <stdio.h>

int main(void) {
    int u, l, s, c, f;
    u = 1000;  l = 0;  s = 10;  f = l;

    while (f <= u) {
        c = 5 * (f - 32) / 9;  // multiply FIRST, then divide
        printf("%d\t%d\n", f, c);
        f = f + s;
    }
    return 0;
}

Big Discovery · Undefined Behavior

printf with no arguments reads stack memory

printf("%d\t%d\n") with no arguments told printf to read two integers from the stack — but none were provided. C doesn't protect you. It reads whatever bytes happen to be there. Random integers within int range (±2³¹) every run. Proved with poc_checkthis.sh — 200+ runs, different values every time.

Lesson: -Wformat= catches this at compile time. Always use -Wall -Wextra.
Big Discovery · Integer Division Truncation

(5/9) is 0 — everything becomes zero

Dividing two integers in C discards the fractional part. 5/9 = 0. Then 0 × anything = 0. Every row printed zero.

Fix: 5 * (f-32) / 9 — multiply first. Or use float/double — covered later.
Error · Cascading Errors

One missing semicolon → everything "undeclared"

Missing ; after the variable declaration caused the compiler to report every subsequent variable as undeclared. The error pointed to the wrong place. Fix the first error first.
